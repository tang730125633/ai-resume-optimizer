# 充值模式设计方案

## 💰 充值 + 余额消费模式

### 核心逻辑

```
用户注册 → 赠送 3 次免费生成模板
    ↓
免费次数用完 → 提示充值
    ↓
充值 50 元 → 账户余额 +50 元
    ↓
使用功能 → 扣除余额
    ↓
余额不足 → 提示充值
```

## 🗄️ 数据库设计

### users 表（用户余额）

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    phone VARCHAR(11) UNIQUE NOT NULL,
    free_count INT DEFAULT 3,              -- 免费生成次数
    balance DECIMAL(10,2) DEFAULT 0.00,    -- 账户余额
    total_recharged DECIMAL(10,2) DEFAULT 0.00,  -- 累计充值
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### orders 表（订单记录）

```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    order_no VARCHAR(32) UNIQUE NOT NULL,
    user_id INT NOT NULL,
    order_type VARCHAR(20) NOT NULL,       -- recharge(充值) / consume(消费)
    amount DECIMAL(10,2) NOT NULL,
    description VARCHAR(200),              -- 订单描述
    payment_method VARCHAR(20),            -- wechat/alipay/balance
    status VARCHAR(20) DEFAULT 'pending',  -- pending/paid/failed
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    paid_at TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 🔄 业务流程

### 流程 1：用户充值

```
1. 用户点击"充值"
2. 选择充值金额（50/100/200/500）
3. 显示收款码（个人码 + 备注手机号）
4. 用户转账
5. 联系客服确认
6. 客服后台操作：
   - 创建充值订单
   - 增加用户余额
   - 订单状态改为 paid
```

**SQL 操作**：
```sql
-- 1. 创建充值订单
INSERT INTO orders (order_no, user_id, order_type, amount, description, status)
VALUES ('RCH20260213150001', 1, 'recharge', 50.00, '充值50元', 'pending');

-- 2. 客服确认后，更新订单和余额
UPDATE orders SET status = 'paid', paid_at = NOW() WHERE order_no = 'RCH20260213150001';
UPDATE users SET balance = balance + 50.00, total_recharged = total_recharged + 50.00 WHERE id = 1;
```

---

### 流程 2：生成模板（免费/付费）

```
1. 用户输入职位 + 年限
2. 点击"生成模板"
3. 检查免费次数：
   - 如果 free_count > 0：扣减免费次数
   - 如果 free_count = 0：检查余额
     - 余额 >= 5 元：扣减 5 元
     - 余额 < 5 元：提示充值
4. 调用 Gemini API 生成
5. 保存到数据库
6. 返回结果
```

**SQL 操作**：
```sql
-- 检查用户状态
SELECT free_count, balance FROM users WHERE id = 1;

-- 情况 1：使用免费次数
UPDATE users SET free_count = free_count - 1 WHERE id = 1;

-- 情况 2：扣除余额
BEGIN;
  -- 创建消费订单
  INSERT INTO orders (order_no, user_id, order_type, amount, description, payment_method, status, paid_at)
  VALUES ('CSM20260213150001', 1, 'consume', 5.00, '生成模板', 'balance', 'paid', NOW());

  -- 扣除余额
  UPDATE users SET balance = balance - 5.00 WHERE id = 1;
COMMIT;
```

---

### 流程 3：优化简历（付费）

```
1. 用户上传简历
2. 点击"优化简历"
3. 检查余额：
   - 余额 >= 10 元：扣减 10 元
   - 余额 < 10 元：提示充值
4. 调用 Claude API 分析
5. 保存到数据库
6. 返回 3 个优化版本
```

**SQL 操作**：
```sql
-- 检查余额
SELECT balance FROM users WHERE id = 1;

-- 扣除余额
BEGIN;
  INSERT INTO orders (order_no, user_id, order_type, amount, description, payment_method, status, paid_at)
  VALUES ('CSM20260213150002', 1, 'consume', 10.00, '优化简历', 'balance', 'paid', NOW());

  UPDATE users SET balance = balance - 10.00 WHERE id = 1;
COMMIT;
```

### 流程 4：导出 PDF（付费）

```
1. 用户点击"导出 PDF"
2. 检查余额：

```

@app.route('/api/user/info', methods=['GET'])
def get_user_info():

    return jsonify(user)

### 2. 创建充值订单

```python
@app.route('/api/recharge/create', methods=['POST'])
def create_recharge_order():
    user_id = request.json['user_id']
    amount = request.json['amount']

    order_no = generate_order_no('RCH')

    db.execute('''
        INSERT INTO orders (order_no, user_id, order_type, amount, description, status)
        VALUES (?, ?, 'recharge', ?, ?, 'pending')
    ''', order_no, user_id, amount, f'充值{amount}元')

    return jsonify({'order_no': order_no, 'qrcode_url': '/static/qrcode.png'})
```

### 3. 客服确认充值（后台接口）

```python
@app.route('/api/admin/confirm-recharge', methods=['POST'])
def confirm_recharge():
    order_no = request.json['order_no']

    # 查询订单
    order = db.query('SELECT user_id, amount FROM orders WHERE order_no = ?', order_no)

    # 更新订单状态
    db.execute('UPDATE orders SET status = "paid", paid_at = NOW() WHERE order_no = ?', order_no)

    # 增加用户余额
    db.execute('''
        UPDATE users
        SET balance = balance + ?, total_recharged = total_recharged + ?
        WHERE id = ?
    ''', order['amount'], order['amount'], order['user_id'])

    return jsonify({'success': True})
```

### 4. 生成模板（扣费）

```python
@app.route('/api/generate-template', methods=['POST'])
def generate_template():
    user_id = request.json['user_id']
    job_title = request.json['job_title']
    years_exp = request.json['years_exp']

    # 查询用户状态
    user = db.query('SELECT free_count, balance FROM users WHERE id = ?', user_id)

    # 判断扣费方式
    if user['free_count'] > 0:
        # 使用免费次数
        db.execute('UPDATE users SET free_count = free_count - 1 WHERE id = ?', user_id)
        cost = 0
    elif user['balance'] >= 5.00:
        # 扣除余额
        order_no = generate_order_no('CSM')
        db.execute('''
            INSERT INTO orders (order_no, user_id, order_type, amount, description, payment_method, status, paid_at)
            VALUES (?, ?, 'consume', 5.00, '生成模板', 'balance', 'paid', NOW())
        ''', order_no, user_id)
        db.execute('UPDATE users SET balance = balance - 5.00 WHERE id = ?', user_id)
        cost = 5.00
    else:
        return jsonify({'success': False, 'error': '余额不足，请充值'}), 400

    # 调用 Gemini API 生成
    template = generate_with_gemini(job_title, years_exp)

    # 保存记录
    db.execute('''
        INSERT INTO resumes (user_id, resume_type, job_title, years_experience, generated_content)
        VALUES (?, 'template', ?, ?, ?)
    ''', user_id, job_title, years_exp, template)

    return jsonify({'success': True, 'template': template, 'cost': cost})
```

@app.route('/api/optimize', methods=['POST'])

## 💡 课程演示要点

**讲解重点**：

1. **为什么用充值模式？**
   - 用户体验好（充一次，多次用）
   - 提高留存率（有余额就会继续用）
   - 降低支付成本

2. **数据库如何记录余额？**
   - `users.balance` 字段
   - `orders` 表记录每笔充值/消费

3. **如何防止余额被刷？**
   - 数据库事务（BEGIN...COMMIT）
   - 先检查余额，再扣费
   - 记录每笔消费订单

4. **个人如何收款？**
   - 阶段 1：个人收款码 + 客服确认
   - 阶段 2：接入第三方支付（码支付）
   - 阶段 3：企业资质 + 官方支付
