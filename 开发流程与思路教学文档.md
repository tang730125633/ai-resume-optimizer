# AI 简历优化器 - 开发流程与思路教学文档

> **课程目标**：让零基础学员理解"为什么要这样做"，而不只是"怎么做"
> **核心理念**：需求驱动技术选型，每一步都有原因

---

## 📋 第一部分：从需求到技术方案

### 1.1 产品需求（用户视角）

我们要做一个 **AI 简历优化器**，核心功能是：

1. ✨ **AI 生成简历模板**（免费功能）
   - 用户输入：职位名称 + 工作年限
   - AI 输出：专业简历模板
   - 限制：每个用户免费 3 次

2. 💎 **AI 深度优化简历**（付费功能）
   - 用户上传：旧简历（PDF/Word）
   - AI 分析：找出问题 + 给建议
   - AI 生成：3 个优化版本
   - 价格：50 元/次

3. 📥 **导出 PDF**
   - 用户可以下载优化后的简历

---

### 1.2 需求分析：为什么需要数据库？

**关键问题来了**：

> "每个用户免费 3 次" —— 你怎么知道这个用户用了几次？

让我们看看不同方案的问题：

#### ❌ 方案 1：用 Session（浏览器会话）
```python
# 用户访问网站时，记录在 session 里
session['count'] = 3
```

**问题**：
- 用户关掉浏览器 → session 清空 → 又有 3 次
- 用户换个浏览器 → 又有 3 次
- 用户清除 Cookie → 又有 3 次
- **结论**：白嫖无限次，无法限制

---

#### ❌ 方案 2：用本地文件
```python
# 把使用次数写到服务器的文件里
with open('user_count.txt', 'w') as f:
    f.write('3')
```

**问题**：
- 多个用户同时访问 → 文件冲突
- 服务器重启 → 文件可能丢失
- 无法区分不同用户
- **结论**：不适合多用户场景

---

#### ✅ 方案 3：用数据库（正确答案）
```python
# 数据库表结构
users (
  id,           # 用户唯一ID
  phone,        # 手机号（用来识别用户）
  free_count,   # 剩余免费次数
  paid_count    # 付费次数
)
```

**优势**：
- ✅ 持久化存储（服务器重启也不丢）
- ✅ 多用户并发安全
- ✅ 可以追踪每个用户的使用情况
- ✅ 支持付费记录查询

---

### 1.3 逻辑链：需求 → 技术选型

```
产品需求：每个用户免费 3 次
    ↓
需要识别"谁是谁"
    ↓

    ↓

    ↓

```

**这就是为什么我们要做数据库设计！**

不是拍脑袋决定的，是被产品需求逼出来的。

## 🗄️ 第二部分：数据库设计

### 2.1 我们需要存什么数据？

根据产品需求，我们需要存：

1. **用户信息**
   - 手机号（用来登录）
   - 免费次数（剩余几次）
   - 付费次数（买了几次）
   - 注册时间

2. **简历记录**
   - 用户上传的原始简历
   - AI 生成的优化版本
   - 生成时间
   - 是否付费

3. **支付订单**
   - 订单号
   - 金额
   - 支付状态
   - 支付时间

---

### 2.2 数据库表设计（用 AI 生成）

**重点**：你不需要会数据库设计，问 AI 就行！

#### 提示词模板：
```
我要做一个 AI 简历优化器，需要设计数据库表结构。

功能需求：
1. 用户可以用手机号登录
2. 每个用户有 3 次免费生成机会
3. 用户可以付费购买优化服务（50元/次）
4. 需要保存用户的简历历史记录

请帮我设计 PostgreSQL 数据库表结构，包括：
- 表名
- 字段名
- 字段类型
- 字段说明
- 表之间的关系
```

#### AI 生成的表结构：

**表 1：用户表（users）**
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,              -- 用户ID（自动递增）
  phone VARCHAR(11) UNIQUE NOT NULL,  -- 手机号（唯一）
  free_count INT DEFAULT 3,           -- 剩余免费次数
  paid_count INT DEFAULT 0,           -- 已购买次数
  created_at TIMESTAMP DEFAULT NOW()  -- 注册时间
);
```

**表 2：简历记录表（resumes）**
```sql
CREATE TABLE resumes (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),   -- 关联用户
  original_content TEXT,              -- 原始简历内容
  optimized_versions JSONB,           -- 优化版本（JSON格式）
  score INT,                          -- AI 评分
  is_paid BOOLEAN DEFAULT FALSE,      -- 是否付费
  created_at TIMESTAMP DEFAULT NOW()
);
```

**表 3：支付订单表（orders）**
```sql
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  user_id INT REFERENCES users(id),
  amount DECIMAL(10,2),               -- 金额
  status VARCHAR(20),                 -- 状态：pending/paid/failed
  pay_time TIMESTAMP,                 -- 支付时间
  created_at TIMESTAMP DEFAULT NOW()
);
```

---

### 2.3 表之间的关系

```
users (用户表)
  ↓ 一对多
resumes (简历记录表)

users (用户表)
  ↓ 一对多
orders (订单表)
```

**解释**：
- 一个用户可以有多条简历记录
- 一个用户可以有多个订单

## 🛠️ 第三部分：完整开发流程（7步法）

### Step 1：需求分析

**做什么**：
- 和 AI 对话，描述你要做什么
- 让 AI 帮你整理成 PRD 文档

**提示词**：
```
我想做一个 AI 简历优化器，目标用户是 30-50 岁的求职者。

核心功能：
1. AI 生成简历模板（免费 3 次）
2. AI 优化旧简历（付费 50 元/次）
3. 导出 PDF

请帮我写一份 PRD 文档，包括：
- 产品定位
- 核心功能
- 技术栈建议
- 开发步骤
```

**产出**：`PRD.md` 文档

---

### Step 2：技术选型

**做什么**：根据需求选择合适的技术

**我们的选择**：
```
前端：HTML + Tailwind CSS（简单、好看）
后端：Python Flask（轻量、易学）
数据库：PostgreSQL（Railway 免费提供）
AI：Claude API + Gemini API（双引擎）
部署：Railway（国内可访问）
```

**为什么这样选**：
- 学员零基础 → 选最简单的
- 国内访问 → 排除被墙的平台
- 免费额度 → 降低学习成本

## 💡 第四部分：开发思路总结

### 4.1 核心思维模式

**传统开发**：
```
学习语法 → 学习框架 → 学习数据库 → 开始做项目
（需要 6 个月）
```

**AI 辅助开发**：
```
有个想法 → 问 AI 怎么做 → 跟着 AI 一步步做 → 项目完成
（需要 1 周）
```

---

### 4.2 遇到问题怎么办？

**万能公式**：

1. **复制错误信息**
2. **问 AI**：
   ```
   我在做 XXX 项目，遇到了这个错误：
   [粘贴错误信息]

   我的代码是：
   [粘贴相关代码]

   请帮我分析原因并给出解决方案
   ```
3. **按 AI 的建议修改**
4. **再次测试**

---

### 4.3 学习路径建议
