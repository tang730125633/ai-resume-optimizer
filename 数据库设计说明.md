## 📋 表详细说明

### 1. users（用户表）

**用途**：存储用户基本信息和使用配额

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | SERIAL | 用户唯一ID | 1 |
| phone | VARCHAR(11) | 手机号（登录凭证） | 13800138000 |
| free_count | INT | 剩余免费次数 | 3 → 2 → 1 → 0 |
| paid_count | INT | 已购买次数 | 0 → 1 → 2 |
| total_generated | INT | 累计生成模板次数 | 统计用 |
| total_optimized | INT | 累计优化简历次数 | 统计用 |
| created_at | TIMESTAMP | 注册时间 | 2026-02-13 10:00:00 |
| last_login | TIMESTAMP | 最后登录时间 | 2026-02-13 15:30:00 |
| is_active | BOOLEAN | 账号是否激活 | true |

**业务逻辑**：
- 新用户注册：`free_count = 3`
- 生成模板：`free_count -= 1`（如果 > 0）
- 付费购买：`paid_count += 1`
- 优化简历：`paid_count -= 1`（如果 > 0）

---

### 2. resumes（简历记录表）

**用途**：存储用户的简历内容和AI处理结果

| 字段 | 类型 | 说明 | 何时有值 |
|------|------|------|----------|
| id | SERIAL | 简历记录ID | - |
| user_id | INT | 关联用户 | - |
| resume_type | VARCHAR(20) | template/optimize | - |
| original_content | TEXT | 原始简历内容 | 优化时 |
| job_title | VARCHAR(100) | 职位名称 | 生成模板时 |
| years_experience | INT | 工作年限 | 生成模板时 |
| ai_analysis | TEXT | AI分析报告 | 优化时 |
| ai_score | INT | AI评分(0-100) | 优化时 |
| generated_content | TEXT | 生成的模板 | 生成模板时 |
| optimized_versions | JSONB | 优化版本数组 | 优化时 |
| is_paid | BOOLEAN | 是否付费 | - |
| is_exported | BOOLEAN | 是否已导出PDF | - |
| pdf_url | VARCHAR(500) | PDF文件URL | 导出后 |

**两种使用场景**：

**场景1：生成模板（免费）**
```json
{
  "resume_type": "template",
  "job_title": "Python后端工程师",
  "years_experience": 5,
  "generated_content": "# 张三\n\n## 个人信息...",
  "is_paid": false
}
```

**场景2：优化简历（付费）**
```json
{
  "resume_type": "optimize",
  "original_content": "用户上传的简历文本...",
  "ai_analysis": "您的简历存在以下问题...",
  "ai_score": 75,
  "optimized_versions": [
    {
      "title": "版本1：传统格式优化",
      "description": "保持原有风格...",
      "content": "优化后的简历内容..."
    },
    {...},
    {...}
  ],
  "is_paid": true
}
```

### 3. orders（支付订单表）

**用途**：记录用户的付费订单

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | SERIAL | 订单ID | 1 |
| order_no | VARCHAR(32) | 订单号（唯一） | ORD20260213150030001 |
| user_id | INT | 关联用户 | 1 |
| resume_id | INT | 关联简历（可选） | 5 |
| amount | DECIMAL(10,2) | 金额 | 50.00 |
| payment_method | VARCHAR(20) | wechat/alipay | wechat |
| status | VARCHAR(20) | pending/paid/failed | paid |
| created_at | TIMESTAMP | 创建时间 | 2026-02-13 15:00:00 |
| paid_at | TIMESTAMP | 支付时间 | 2026-02-13 15:05:00 |
| expired_at | TIMESTAMP | 过期时间 | 2026-02-13 15:30:00 |

**订单状态流转**：
```
pending（待支付）
   ↓ 用户扫码支付
paid（已支付）→ 增加 paid_count
   ↓ 超时未支付
failed（支付失败）
   ↓ 用户申请退款
refunded（已退款）
```

---

### 4. activity_logs（操作日志表）

**用途**：记录用户关键操作，用于数据分析和问题排查

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | SERIAL | 日志ID | 1 |
| user_id | INT | 用户ID | 1 |
| action | VARCHAR(50) | 操作类型 | generate/optimize/export |
| details | JSONB | 操作详情 | {"job_title": "工程师"} |
| ip_address | VARCHAR(45) | IP地址 | 192.168.1.1 |
| user_agent | TEXT | 浏览器信息 | Mozilla/5.0... |
| created_at | TIMESTAMP | 操作时间 | 2026-02-13 15:00:00 |

**操作类型**：
- `login` - 用户登录
- `generate` - 生成模板
- `optimize` - 优化简历
- `export` - 导出PDF
- `pay` - 支付订单

## 🔄 业务流程示例

### 流程1：新用户生成模板

```sql
-- 1. 用户登录（手机号：13800138000）
INSERT INTO users (phone) VALUES ('13800138000')
RETURNING id, free_count;  -- 返回：id=1, free_count=3

-- 2. 生成模板
INSERT INTO resumes (user_id, resume_type, job_title, years_experience, generated_content)
VALUES (1, 'template', 'Python工程师', 5, 'AI生成的简历内容...');

-- 3. 扣减免费次数
UPDATE users SET free_count = free_count - 1 WHERE id = 1;

-- 4. 记录日志
INSERT INTO activity_logs (user_id, action, details)
VALUES (1, 'generate', '{"job_title": "Python工程师"}');
```

---

### 流程2：付费优化简历

```sql
-- 1. 检查用户是否有付费次数
SELECT paid_count FROM users WHERE id = 1;  -- 返回：0（需要购买）

-- 2. 创建订单
INSERT INTO orders (order_no, user_id, amount, status, expired_at)
VALUES ('ORD20260213150030001', 1, 50.00, 'pending', NOW() + INTERVAL '30 minutes')
RETURNING id;  -- 返回：order_id=1

-- 3. 用户支付成功（回调接口）
UPDATE orders SET status = 'paid', paid_at = NOW() WHERE id = 1;
UPDATE users SET paid_count = paid_count + 1 WHERE id = 1;

-- 4. 优化简历
INSERT INTO resumes (user_id, resume_type, original_content, ai_analysis, ai_score, optimized_versions, is_paid)
VALUES (1, 'optimize', '原始简历...', 'AI分析...', 85, '[...]', true);

-- 5. 扣减付费次数
UPDATE users SET paid_count = paid_count - 1 WHERE id = 1;
```

## 📈 数据统计查询

### 运营数据

```sql
-- 今日新增用户
SELECT COUNT(*) FROM users WHERE DATE(created_at) = CURRENT_DATE;

-- 今日生成模板次数
SELECT COUNT(*) FROM resumes
WHERE resume_type = 'template' AND DATE(created_at) = CURRENT_DATE;

-- 今日付费金额
SELECT SUM(amount) FROM orders
WHERE status = 'paid' AND DATE(paid_at) = CURRENT_DATE;

-- 用户留存率（7日）
SELECT
  COUNT(DISTINCT user_id) as active_users
FROM activity_logs
WHERE created_at >= CURRENT_DATE - INTERVAL '7 days';
```

### 用户分析

```sql
-- 付费用户占比
SELECT
  COUNT(CASE WHEN paid_count > 0 THEN 1 END) * 100.0 / COUNT(*) as pay_rate
FROM users;

-- 平均每用户生成次数
SELECT AVG(total_generated) FROM users;

-- 高价值用户（付费3次以上）
SELECT phone, paid_count, total_optimized
FROM users
WHERE paid_count >= 3
ORDER BY paid_count DESC;
```

## 🛠️ 在 Railway 中创建数据库

### 步骤1：创建 PostgreSQL 数据库

1. 登录 Railway
2. 点击 "New Project" → "Provision PostgreSQL"
3. 等待数据库创建完成

### 步骤2：连接数据库

```bash
# Railway 会自动提供连接信息
DATABASE_URL=postgresql://user:password@host:port/database
```

### 步骤3：执行 SQL 创建表

**方法1：使用 Railway 控制台**
- 在 Railway 项目中点击 PostgreSQL
- 点击 "Data" 标签
- 点击 "Query" 按钮
- 粘贴 `database_schema.sql` 内容
- 点击 "Run"

**方法2：使用命令行**
```bash
# 安装 psql 客户端
brew install postgresql

# 连接数据库
psql $DATABASE_URL

# 执行 SQL 文件
\i database_schema.sql

# 查看表
\dt

# 退出
\q
```
